 ## 돈과 행복의 관계를 학습하는 AI

> **v2 → v3 핵심 변경**
> 
> 1. **결제 직후 점수 입력 제거** — 사용자 부담 감소
> 2. **결제 전 UX가 카드형 → 챗봇 상담으로 전환** — 풍부한 상호작용
> 3. **데이터 입력 채널을 2개로 단순화**: 결제 전 챗봇 + 주 1회 회고
> 4. 챗봇 톤: **담담한 상담사** (공감형 친구가 아님)

---

# 1. Product Definition

## 1.1 핵심 문제

20대 사회초년생은:

- 처음으로 자유롭게 쓸 돈이 생겼고
- 소비 통제 욕구는 있지만
- 가계부는 너무 귀찮고
- 절약 앱은 "쓰지 마"만 외쳐서 죄책감 유발

→ 이들이 원하는 건 *"쓰지 마"가 아니라 "다음엔 더 잘 쓸 수 있어"*. → 행복으로 수정해서 설명하면 좋을 듯

## 1.2 제품 정의

> **Aftertaste는 결제 전 챗봇 상담과 주 1회 회고를 통해, 사용자의 만족 패턴을 학습하고, "이 돈이면 어디에 쓰는 게 더 행복할지"를 데이터로 보여주는 AI 서비스이다.**
> 

## 1.3 핵심 가치

후회 줄이기 (X) → **더 행복한 대안 찾기** (O)

- 메시지: "다른 행복도 있어요"
- 스탠스: 발견과 제안 (경고/제한 X)
- 사용자 감정: 호기심/안도 (죄책감 X)
- 비유: **행복 판단 설문지**

# 2. Target Users

## 2.1 Primary Target

**20대 사회초년생** — 소득은 있지만 소비 통제가 어려운 시기.

### 페르소나

- 27세 직장인, 월 실수령 250만원
- 배달/카페/쇼핑에서 "왜 또 샀지" 자주 경험
- 가계부 앱 3주 만에 안 봄
- 절약 앱의 죄책감 유발이 싫음

# 3. Core User Flows

이 앱은 **2개의 시점** 에서만 작동한다.

```
[결제 전]                          [매주 일요일]
챗봇 상담                           시각적 회고
"이거 사면 어떨까?"                  "한 주의 행복 곡선"
   │                                   │
   └───── 결정/요약 ─────► 학습 데이터 ◄──── 점수 입력
```

## 3.1 Flow A: 결제 전 챗봇 상담 (핵심 기능)

### 트리거

사용자가 자율 소비(패션/전자제품/취미 등)를 고민할 때, 앱 열고 챗봇에 입력.

### 챗봇 톤: **담담한 상담사**

- 공감형 친구 (X) — "헐 그거 사야지!"
- 소크라테스 (X) — 끝없는 반문
- 심리상담 (X) — "지금 기분이 어떠세요?"
- **데이터 기반 상담사 (O)** — "지난 패턴을 보시면 이런 경향이 있어요"

### 대화 예시

> **사용자**: 에어팟 프로 35만원 살까 고민 중
> 
> 
> **AI**: 35만원이군요. 지난 한 달간 비슷한 금액대 소비를 보시면, 여행이 만족도 4.8, 강의가 4.6으로 상위였어요. 이번 35만원은 어떤 결을 원하세요?
> 
> **사용자**: 출퇴근 때마다 쓰니까 자주 쓸 것 같아
> 
> **AI**: 자주 쓰는 물건이라는 점은 만족도와 상관관계가 높습니다. 다만 비슷한 가전 카테고리에서 지난 3개월 결제 이력 중 한 건은 "다시 살 의향: 애매하다"로 답하셨네요.
> 
> [살게요 / 다시 생각 / 안 살게요]
> 

### 종료 방식

**명시적 3개 버튼**으로 종료:

- ✅ 살게요
- 🤔 다시 생각
- ❌ 안 살게요

→ 결정 자체가 학습 데이터.

### 대화 저장 방식

**AI가 요약한 핵심 몇 줄만**:

```
요약 예시:
- 상품: 에어팟 프로 35만원
- 사용자 사유: 자주 사용 예정
- AI 제시: 만족도 상위 카테고리 비교 + 가전 카테고리 과거 이력
- 결정: 살게요
```

→ 처리 가벼움 + 회고 시 활용 가능

### 콜드 스타트 (데이터 부족 시)

1. 신규 사용자는 오픈뱅킹 연동 후 **과거 1개월 데이터 라벨링**부터
2. "이중에 만족스러웠던 건?" 라벨 5개 이상 쌓이면 챗봇 활성화
3. 라벨 부족 시 챗봇은 일반 통계로 안내 ("자주 쓰는 물건은 평균적으로 만족도가 높은 편이에요")

## 3.2 Flow B: 일요일 저녁 회고 (주 1회)

### 시점

일요일 저녁 — 한 주 정리하는 의식적 시간.

### 회고 대상 선정 (AI 자동)

사용자가 개수 정하지 않고 AI가 그 주 소비량에 따라 조절.

선정 기준 (4가지 믹스):

1. **만족 강화**: 이미 만족도 높았던 항목 → 추천 정확도 향상
2. **공백 메우기**: 아직 안 물어본 큰 금액 소비
3. **Active Learning**: AI가 헷갈리는 소비
4. **다양성**: 카테고리 골고루

### 입력 방식

- **1-5 점수** (필수)
- **한 줄 텍스트** (선택)

### 챗봇 상담했던 항목과의 연결

- 결제 전 챗봇에서 "살게요"한 항목 → 회고에 **우선 등장**
- 챗봇 요약("자주 쓸 거라서 샀음")이 회고 화면에 함께 표시
- → "그때 예상 vs 지금 만족도" 비교 가능

---

# 4. 카테고리 분류 시스템

회고에서만 사용 (결제 직후 입력 없으니까).

| 분류 | 예시 | 회고 처리 |
| --- | --- | --- |
| **즉시 소비형** | 배달, 카페, 술 | 회고에서 1-5 점수 |
| **지속 소비형** | 옷, 가전, 구독, 강의 | 회고에서 1-5 점수 + "지금 잘 쓰고 있는지" |
| **필수 소비형** | 교통, 통신, 공과금 | 회고 대상에서 제외 |

### 챗봇 상담에서의 활용

- AI가 입력 카테고리 자동 분류해 적합한 통계 제시
- 즉시 소비형 입력 시: "비슷한 즉시 소비는 시간 지나면 만족도가 평균 X점 떨어지는 경향이 있어요"
- 지속 소비형 입력 시: "지속 소비형은 지금 점수보다 1주 후 점수가 진짜 만족도예요"

# 6. 톤 & 매너

## 6.1 컨셉

**"행복 판단 설문지"** — 따뜻하고 고요.

## 6.2 챗봇 카피 가이드

| ✅ 좋음 | ❌ 나쁨 |
| --- | --- |
| "지난 패턴을 보시면 이런 경향이 있어요" | "와 그거 좋죠 사세요!" |
| "비슷한 카테고리 만족도는 평균 3.8입니다" | "후회할 거예요 사지 마세요" |

## 6.3 디자인 톤

- 따뜻한 베이지/세이지 색조
- 차분한 산세리프
- 미니멀 타이포그래피 중심
- 정보 시각화는 절제된 그래프

# 7. AI System Design

## 7.1 데이터 입력 (단순화됨)

- 거래 내역 (오픈뱅킹)
- **챗봇 대화 요약** (AI 자동 생성)
- **챗봇 결정** (살게요/다시 생각/안 살게요)
- **주간 회고 점수** (1-5)
- 회고 텍스트 (선택)

## 7.2 AI의 3가지 역할

### 7.2.1 분류기

- 거래 → 3카테고리 분류
- 챗봇 입력 → 카테고리 추정

### 7.2.2 챗봇 상담 엔진

- 사용자 입력 분석
- 과거 데이터에서 관련 통계 추출
- 담담한 톤으로 정보 제시
- 대화 핵심 요약 생성

### 7.2.3 회고 큐레이터

- 어떤 소비를 회고에 올릴지 선정
- Active learning 원리

## 7.3 핵심 원칙

> **AI는 답을 주는 것이 아니라, 데이터를 보여주고 결정을 돕는다.**
>

# 8. Onboarding Flow

## Step 1. 오픈뱅킹 연동

첫 화면 — 빠른 가치 제공.

## Step 2. 과거 3개월 회고

- "이중에 만족스러웠던 건?" → 결제 패턴을 바탕으로 질문(지속적 구매, 큰 지출 등등)
- 5개 이상 라벨링 → 챗봇 활성화

## Step 3. 첫 인사이트 카드

- "당신은 ___에 쓸 때 만족도가 높네요"

## Step 4. 챗봇 첫 상담 유도

- "고민되는 소비가 있으면 물어보세요"
- 첫 상담 경험

# 9. Success Metrics

## 9.1 North Star Metric

> **챗봇 상담에서 "안 살게요" + "다시 생각"으로 이어진 후, 1주 내 미결제 비율**
> 

### 왜 이 지표인가

- 제품이 실제로 *행동 변화*를 만들었는지 측정
- 단순 사용량보다 임팩트
- "관점 전환" 이라는 핵심 가치를 직접 측정

## 9.2 Secondary Metrics

- 주간 회고 완료율
- 챗봇 사용 횟수 (능동 진입)
- 챗봇 상담 평균 길이 (engagement)
- "안 쓴 돈 카운터" 누적 금액
- 월간 평균 만족도 추세

# 12. Risks & Mitigation

| 리스크 | 영향 | 대응 |
| --- | --- | --- |
| **콜드 스타트** | 높음 | 온보딩 1개월 회고 강제, 5라벨 후 챗봇 활성화 |
| **챗봇 사용 안 함** (능동 진입 부담) | 매우 높음 | 결제 알림 와이드 진입, "고민 중인 소비 있으세요?" 푸시, 첫 5회 풀기능으로 강한 첫인상 |
| **챗봇 톤이 부담** | 중간 | "담담한 상담사" 톤 일관성 유지, 사용자 톤 조절 옵션 |
| **회고 1주 후 갭 데이터 부재** | 중간 (v2에서 잃은 차별화) | 챗봇 요약 ↔ 회고 연결로 보완 |
| **다운그레이드 불만** | 중간 | 부분 복귀 보너스, A/B 테스트 |

# 13. v2 대비 트레이드오프 정리

**얻은 것**:

- 사용자 부담 절반으로 감소 (입력 채널 3 → 2)
- 챗봇으로 풍부한 상호작용
- 결정 시점 직접 개입 (결제 직후 사후 점수보다 강한 효과)

**잃은 것**:

- "결제 직후 vs 시간 후" 갭 데이터 (v2의 핵심 차별화 자산)

**보완 전략**:

- 챗봇 상담 요약 ↔ 회고 점수 연결
- "그때 예상 vs 실제 만족도" 비교 가능
- 챗봇이 시간차 만족도 통계를 직접 제시 ("즉시 소비형은 1주 후 평균 X점 하락")

# 15. Open Questions

1. **챗봇 진입 환기**: 사용자가 능동 진입하지 않을 때 어떻게 트리거할지? (푸시 vs 결제 알림 연동)
2. **챗봇 응답 시간**: LLM 호출 비용 vs UX 즉시성
3. **요약 정확도**: AI 요약이 회고에 잘못 연결되면 사용자 신뢰 잃음 — 검증 필요
4. **다운그레이드 모델 수용도**: A/B 테스트 설계 필요
5. **첫 인사이트 카드**: 데이터 부족할 때 어떤 메시지로 첫 가치 전달할지
